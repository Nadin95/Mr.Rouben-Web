Propuesta: Opcion 3 (Hosting Node economico + WhatsApp API oficial + OCR externo)
Fecha: 2026-04-15
Region objetivo: San Francisco, Cordoba, Argentina

OBJETIVO
Implementar una arquitectura economica y estable para produccion, manteniendo el codigo base actual (TypeScript + Express + MongoDB) y agregando servicios externos para mensajeria oficial, OCR de comprobantes y almacenamiento de imagenes.

--------------------------------------------------------------------
1) ESTRATEGIA POR ETAPAS (RECOMENDADA)
--------------------------------------------------------------------
ETAPA 1 - Base productiva estable
- Deploy en hosting Node economico (Render o Railway).
- Base de datos MongoDB Atlas.
- Dominio + SSL + variables de entorno seguras.

ETAPA 2 - Canal oficial de WhatsApp
- Migrar de integracion legacy a Meta WhatsApp Cloud API.
- Webhook oficial para recepcion y envio de mensajes.
- Plantillas de mensajes (aprobadas por Meta) para notificaciones.

ETAPA 3 - OCR de comprobantes
- Integrar Google Cloud Vision API.
- Leer texto de comprobantes automaticamente.
- Sugerir validacion al admin (con fallback manual si la confianza es baja).

ETAPA 4 - Operacion y monitoreo
- Uptime y alertas basicas.
- Reintentos de integraciones externas.
- Logs de trazabilidad por pedido/comprobante.

3) EXPLICACION DE CADA SERVICIO (PARA CLIENTA)
--------------------------------------------------------------------
1. Hosting Node (Render o Railway)
- Que es: donde vive y se ejecuta la pagina web.
- Para que sirve: mantiene el sitio online 24/7.
- Por que conviene: costo bajo para empezar y escalable cuando crece.

2. MongoDB Atlas
- Que es: base de datos administrada en la nube.
- Para que sirve: guarda usuarios, pedidos, productos y comunidad.
- Por que conviene: evita administrar servidor de base de datos manualmente.

3. WhatsApp Cloud API oficial (Meta)
- Que es: canal oficial de WhatsApp Business.
- Para que sirve: enviar/recibir mensajes de forma estable y profesional.
- Por que conviene: mas confiable que soluciones no oficiales; apto para negocio.

4. Google Cloud Vision OCR
- Que es: servicio de lectura de texto en imagenes.
- Para que sirve: extrae datos del comprobante de pago automaticamente.
- Por que conviene: reduce revision manual y acelera validacion.

5. Cloudinary (storage de imagenes)
- Que es: almacenamiento y distribucion de imagenes.
- Para que sirve: guardar fotos de productos, carrusel y comprobantes.
- Por que conviene: evita depender del disco local del hosting.

6. Brevo (email transaccional)
- Que es: plataforma de envio de emails.
- Para que sirve: verificacion de cuenta, notificaciones y comunicaciones operativas.
- Por que conviene: mejor entregabilidad que usar correo personal SMTP.

7. UptimeRobot (monitoreo)
- Que es: servicio de monitoreo de disponibilidad.
- Para que sirve: avisa si la pagina se cae o responde lento.
- Por que conviene: deteccion temprana de incidentes con costo muy bajo.

--------------------------------------------------------------------
4) COMPATIBILIDAD CON EL STACK ACTUAL

--------------------------------------------------------------------
Para equilibrio costo/estabilidad:
